# Prompts — Research & Analysis

For pre-call preparation, competitive research, and helping clients make informed decisions.

## Competitor / market research

### RA-01: Competitor snapshot
```
My client runs [BUSINESS TYPE] in [LOCATION / MARKET]. They want to know their top 5 competitors.

What I know about the client:
[BRIEF]

Research and return a competitor snapshot for each of 5 likely competitors:
- Name
- Positioning (in one sentence)
- Pricing range (if public)
- Their visible differentiator (what they emphasize)
- Weakness or gap (what they DON'T do well — based on public signals)

Be honest about what's guessable vs unknown. Don't invent pricing or metrics.
```

### RA-02: Competitive message audit
```
My client wants to know how 3 competitors position themselves.

Competitors:
1. [NAME + WEBSITE]
2. [NAME + WEBSITE]
3. [NAME + WEBSITE]

For each, based on their website copy / public messaging, extract:
- Headline / primary pitch
- Top 3 benefits they emphasize
- Their implied target customer
- Any promise language
- Their CTA style (soft vs hard)

Return as a comparison table so my client can see overlaps and gaps.
```

### RA-03: Gap analysis for positioning
```
My client is [BUSINESS] in [MARKET]. Based on the competitive landscape research below, identify 3-5 positioning gaps — spaces competitors AREN'T occupying that my client could own.

Competitive research:
[PASTE]

For each gap:
- The gap (a specific underserved customer, problem, or angle)
- Why competitors don't occupy it
- What my client would need to credibly claim it
- A draft positioning statement

Be specific. Don't suggest "premium quality" or "great customer service" — those aren't real positioning.
```

## Prospect research (pre-call)

### RA-04: Prospect profile from LinkedIn
```
Build a prospect profile from this LinkedIn info:
[PASTE]

Return:
- Role + seniority
- Career pattern (how they got here, what they seem to care about)
- Tenure pattern (long-term or movers?)
- Industry specialization
- Content patterns (what do they post about? what do they engage with?)
- Likely decision-making authority for [TOPIC]
- Likely KPIs / pressures
- Best angle to open with

Length: 250 words, bulleted.
```

### RA-05: Company research brief
```
Research [COMPANY NAME] for an upcoming sales call.

Return:
- Elevator pitch of the company (2 sentences)
- Size indicators (employees, revenue if public, growth signals)
- Recent news / announcements (last 6 months)
- Likely pain points for [MY SERVICE TYPE]
- Who the buyer probably is (role + name if findable)
- Any red flags (layoffs, lawsuits, leadership changes)

If info is behind paywalls or not public, say "unknown" rather than guess.
```

## Industry / market overview

### RA-06: Industry landscape brief
```
I need a 500-word industry landscape for [INDUSTRY] to help a client decide on positioning.

Cover:
- Current state (maturity, growth rate if known, key players)
- Recent major trends (2024-2026)
- Emerging threats (regulation, tech shift, customer behavior)
- Opportunities for a small player
- What's overhyped vs what's actually working

Be candid. Flag anything where I'm making claims I can't substantiate.
```

### RA-07: Customer persona development
```
My client serves [TARGET CUSTOMER TYPE]. Build 3 detailed buyer personas.

For each:
- Name + 1-line description
- Demographics (age range, income, location type, job)
- Their typical day (10 sentences)
- Top 3 frustrations
- Top 3 goals
- Where they go for info (specific publications, influencers, communities)
- Objections they'd have to buying from my client
- Messaging that would resonate

Don't default to generic personas. Pull from real patterns in [INDUSTRY].
```

## Data analysis (light)

### RA-08: Spreadsheet interpretation
```
Below is a CSV of [DATA TYPE — e.g., monthly sales by product].

[PASTE CSV OR SUMMARY]

Analyze and return:
- Top-line observation (1 sentence)
- 3-5 patterns worth flagging
- 2-3 anomalies or outliers (if any)
- 2-3 questions the data raises but doesn't answer
- A recommended next cut or next question to investigate

Don't invent data that isn't there. If the data is insufficient for a given conclusion, say so.
```

### RA-09: Survey response synthesis
```
My client ran a survey. Below are the open-ended responses to: [QUESTION].

Responses:
[PASTE]

Synthesize:
- Top 5 themes (with count / percentage)
- Most representative quote for each theme (direct, unedited)
- Surprise themes (mentioned by <5% but interesting)
- Recommendations based on patterns
- What's missing (questions the survey didn't ask but should have)

Length: under 500 words.
```

## Content/editorial research

### RA-10: Topic research for a blog post
```
I'm writing a blog post on [TOPIC] for [CLIENT]. Audience: [BRIEF].

Research and return:
- 3 angles I could take on this topic
- Key stats or data points (only cite real sources — don't invent)
- 3-5 expert perspectives / quotes I could reference (with sources)
- Common myths/misconceptions about this topic
- What the competing content is saying (and NOT saying)

Flag anything you're unsure about.
```

### RA-11: Question research (for an interview)
```
My client is interviewing [EXPERT] on their podcast about [TOPIC].

Based on what I know:
- Expert's background: [BRIEF]
- Topic: [SPECIFIC]
- Audience: [BRIEF]

Generate:
- 10 substantive interview questions
- 3 "go deeper" follow-ups for each
- 2 questions that might be uncomfortable but worth asking
- 1 question that will probably go unexpectedly long (and is worth letting run)

Avoid questions they've probably been asked 100 times.
```

## Client decisions

### RA-12: Decision framework
```
My client is weighing a decision: [DECISION].

Options they're considering:
A) [OPTION A]
B) [OPTION B]
C) [OPTION C]

Known considerations:
[PASTE]

Build a decision matrix:
- Criteria that matter (5-7, weighted if client has preferences)
- Score each option against each criterion (1-5, with brief reasoning)
- Total weighted score per option
- A "watch for this" list of risks for the top-ranked option
- A recommendation + 1 question that could change it

Don't be wishy-washy. Make a call.
```

### RA-13: Pros and cons, honestly
```
The client is considering: [CHOICE].

Generate an honest pros/cons analysis:
- 5 pros (with realistic magnitude — don't inflate)
- 5 cons (with realistic magnitude — don't underplay)
- The "if only one thing matters" angle
- What I'd ask the client to help them decide

No corporate hedging. Say what you actually think.
```

## Reading comprehension support

### RA-14: Summarize a long document
```
Summarize this [DOCUMENT TYPE — contract / report / article / book chapter] for my client.

Document:
[PASTE OR URL + CONTEXT]

Return:
- TL;DR (3-5 sentences)
- Key sections (what each section covers, not a retelling)
- Decisions or obligations (what the document asks of the reader)
- Flags (anything unusual, risky, or worth discussion)
- Questions the document raises

Length: fits on one screen.
```

### RA-15: Contract review (non-legal)
```
⚠️ This is NOT legal review. I am not an attorney. This is plain-English summary support only.

Below is a contract my client is considering. Summarize in plain English:
- What my client is agreeing to do
- What the other party is agreeing to do
- Payment terms
- Termination conditions
- Any unusual or potentially problematic clauses (flag, don't interpret)

Tell the client: "Have an attorney review before signing, especially [X, Y, Z specific items]."

Contract:
[PASTE]
```

---

## Research principles

- Distinguish what you know, what you've inferred, and what you're guessing. Use "it appears," "likely," "confirmed" deliberately.
- Don't invent stats. If you can't find a real source, say "not readily available" and note what to search for.
- Short reports that get read beat comprehensive reports that don't.
- Your job is to make the client smarter, not to show off.
