# Prompts — Product Descriptions

Honest descriptions that build trust and sell. Work with ES-30 through ES-34 titles.

## ES-40: Standard eBay description

```
Write a complete eBay listing description for this item.
[ITEM NAME + KEY DETAILS]
[CONDITION DETAILS — be specific: small chip on lid, minor utensil
marks, tag cut out of collar, hairline crack under glaze, etc.]
[DIMENSIONS / SIZE / WEIGHT]
[ERA OR YEAR IF KNOWN]
[PROVENANCE IF INTERESTING — "from an estate of a collector", etc.]

Structure the description:
1. Opening line (1-2 sentences — what this is and why it's special,
   no hype)
2. Detailed condition report — honest, specific, mentions any flaws
   first (buyer trust depends on this)
3. Measurements / specs in clear format
4. Shipping note (if heavy/fragile, mention careful packing)
5. Returns policy summary (what you offer — 30-day, as-is, etc.)

Rules:
- No exclamation points except at most one per listing
- No spammy words ("RARE!!!", "L@@K", "MUST SEE")
- Buyer trusts an honest description over a hyped one
- Include 1-2 sensory details (how it feels, sounds, weighs) that
  a photo can't convey
- Close with a soft call-to-action ("Message me with any questions")
```

## ES-41: Description for a collector-grade item

```
This item appeals to serious collectors of [CATEGORY — e.g.,
"vintage hand planes", "mid-century Pyrex", "pre-1970 Stanley
tools"]. My buyer cares about:
- Provenance
- Completeness (all original parts, markings intact)
- Specific era / manufacturing details
- Condition in collector terms (not user-grade terms)

Item: [DETAILED ITEM DESCRIPTION INCLUDING ALL MARKINGS, era
indicators, condition in collector-specific language]

Write a collector-targeted description:
- Lead with the collector-important details (era, markings,
  completeness)
- Use correct terminology that signals you know what you have
- Note any repairs, restorations, or sympathetic cleaning honestly
- Mention where it came from if it adds provenance (estate sale
  from a long-time collector, etc.)
- Offer to ship via tracked/insured methods for high-value items

Length: 200-400 words, buyer-dense, no fluff.
```

## ES-42: Description for vintage clothing with measurements

```
Vintage clothing item:
- Type, era, maker as described in my title
- Flat measurements (don't convert to modern sizes in the body — just
  report actual measurements):
  - Shoulder to shoulder: [X]
  - Bust / chest (pit to pit): [X]
  - Waist (laid flat): [X]
  - Hip (widest): [X]
  - Length (shoulder to hem OR back collar to hem): [X]
  - Sleeve (shoulder seam to cuff): [X]
- Fabric content: [FROM TAG OR BEST GUESS]
- Care instructions: [IF READABLE]
- Condition: [DETAILED — ANY flaws, stains, repairs, smoke/pet-free
  storage]

Write the description. Vintage clothing buyers are hyper-focused on
fit — measurements are the most important content. Structure:
1. 2-3 sentences: what this is + its vibe (cocktail dress, workwear,
   etc.)
2. Measurements block (clearly labeled, easy to scan)
3. Condition report
4. Shipping / returns note
```

## ES-43: Honest description of a flawed-but-valuable item

```
I have an item that's genuinely valuable but has noticeable flaws.
Most resellers would hide the flaws in photos or sell as "parts
only" cheap. I want to sell it at a fair discounted price to a
buyer who understands what they're getting.

Item: [DESCRIPTION]
Flaws: [DETAILED LIST OF EVERY FLAW]
What a perfect example sells for: $[PRICE RANGE]
My target price: $[DISCOUNTED PRICE]

Write a listing description that:
- Leads with what's great about it (era, pattern, rarity — whatever
  makes it valuable)
- Clearly states the flaws with specifics (not "some wear" — "1/4"
  chip on rim at 2 o'clock position")
- Explains why I'm pricing it at $[PRICE] vs the premium range
- Targets the buyer who WANTS an affordable-but-imperfect version
  (filling a collection, using daily instead of displaying)
- Sets expectations so there are no returns

Tone: honest, respectful of buyer's intelligence, no hype.
```

## ES-44: "As-is / untested" description for items I can't verify

```
I bought this item at an estate sale without the ability to fully
test or authenticate it. I want to sell it honestly as "as-is" so
the buyer understands the risk.

Item: [DESCRIPTION]
What I CAN verify: [WHAT YOU'VE CONFIRMED — e.g., "item is clean,
no visible cracks, no obvious repairs"]
What I CAN'T verify: [WHAT'S UNKNOWN — "haven't tested electrical
function", "haven't authenticated signatures", etc.]
Condition: [AS-FOUND CONDITION]

Write an "as-is" description that:
- Describes what I can confirm
- Clearly lists what I can't confirm and why
- Sets a "sold as-is, no returns" policy
- Prices in line with that (discounts versus verified items)
- Recommends the buyer do their own research before bidding

Tone: transparent and low-pressure. Buyer should leave the listing
knowing exactly what they're getting into.
```
