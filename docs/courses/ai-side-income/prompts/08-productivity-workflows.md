# Prompts — Productivity & Personal Workflows

For your own freelance practice: task planning, email management, decision-making, and keeping yourself organized.

## Daily planning

### PW-01: Daily priority triage
```
Here's everything on my plate for today:
[DUMP EVERYTHING — clients, tasks, errands, calls, anything]

Triage into:
- MUST DO TODAY (3 items max)
- Should do today (4-6 items)
- Could wait (everything else)
- Drop / delegate / automate (honest flags)

For each MUST DO, estimate time + note what success looks like.
Push back on me if I've listed too many "must do"s — I know it's a pattern.
```

### PW-02: Morning startup routine
```
It's Monday. Here's what I have on the go:
- Active client projects: [LIST]
- Pending proposals: [LIST]
- Outreach in motion: [LIST]
- Personal stuff: [LIST]

Generate my week's game plan:
- 3 outcomes that would make this a great week
- Morning routine (30 min) for each weekday
- Theme per day (client delivery? sales? admin? learning?)
- 1 no-meetings block per day

Realistic, not aspirational. I'm part-time, not grinding 60 hours.
```

### PW-03: End-of-day shutdown
```
It's the end of the day. Here's what I did / didn't get to:
[BRAIN DUMP]

Generate:
- What to celebrate (genuinely — even small wins)
- 1 unfinished item that needs to be top of tomorrow
- 1 item I can let go of (cross off, not move)
- A 1-sentence note I can send my future-self about where I left off
- A "close the laptop" prompt — what to stop worrying about until morning

Keep it warm. I've worked hard.
```

## Email management

### PW-04: Inbox triage
```
Below are the subject lines + 1-line previews of [N] unread emails.

[PASTE]

Triage into:
- REPLY TODAY (urgent or short)
- REPLY THIS WEEK
- READ LATER (newsletters, articles)
- UNSUBSCRIBE (no longer adding value)
- DELETE / ARCHIVE (done, no action needed)

For REPLY TODAY, suggest a 1-sentence direction for each.
```

### PW-05: Draft response to a difficult email
```
I need to reply to this email. I've been avoiding it because [WHY — frustrated, unclear, big ask, etc.].

Email:
[PASTE]

Draft 2 possible responses:
- Response A: "just get it done" version (short, direct, closes the loop)
- Response B: "handle with care" version (more context, relationship-preserving)

Also note: what's my actual fear about responding? (1 sentence — real, not performative)
```

### PW-06: Declining a request gracefully
```
Someone asked: [REQUEST]

I need to say no because [REAL REASON].

Draft a response that:
- Declines clearly (no "unfortunately" buildup)
- Doesn't over-explain
- Offers an alternative if appropriate (referral, different time, scaled-down version)
- Preserves the relationship

Length: under 100 words. Tone: warm but firm.
```

## Project planning

### PW-07: Breaking a project into steps
```
I just got hired for: [PROJECT]

Agreed scope:
[DETAILS]

Break into concrete tasks:
- Setup phase (what needs to happen before I can start)
- Execution phase (core work, with sub-tasks)
- Review & delivery phase
- Invoicing

For each, estimate hours (be realistic — I'm [EXPERIENCE LEVEL]).

Then: propose a timeline backward from the deadline, with buffer.
```

### PW-08: Project post-mortem
```
Just wrapped [PROJECT] for [CLIENT].

Rough summary:
- Scope: [BRIEF]
- Final hours: [ESTIMATE]
- Outcome: [GOOD / OK / BAD]
- Surprises (good or bad): [LIST]

Generate a 1-page post-mortem:
- What went well (specific)
- What went poorly (specific, non-blaming)
- What I'll do differently next time (2-3 concrete actions)
- How this affects my pricing or process going forward
- Whether I'd take a similar project again (yes / maybe / no)

Honest. Not therapy — just operational review.
```

## Decision support

### PW-09: Is this a yes or no?
```
Someone's offering: [OPPORTUNITY]

My gut: [LEAN YES / LEAN NO / TRULY NEUTRAL]

Force me through a decision:
- What's the best case if I say yes?
- What's the worst case if I say yes?
- What's the best case if I say no?
- What's the worst case if I say no?
- What's the "hell yes" bar this would need to clear?
- Does this opportunity pass the "hell yes" bar?

Then give me a recommendation. Be blunt.
```

### PW-10: Should I raise my rates?
```
Here's my current situation:
- Current rate: $[X]
- Months at this rate: [N]
- Demand signals: [FULL CALENDAR / MIXED / SLOW]
- Recent wins / testimonials: [BRIEF]
- Reasons I'm considering raising: [PASTE]
- Reasons I'm hesitant: [PASTE]

Analysis:
- Based on the signals, is a rate increase reasonable? (objective)
- What's a realistic increase? (not "industry average" — specific to me)
- Which clients get grandfathered vs. which get the new rate? (framework)
- What's the actual risk (losing clients)?
- Script for announcing it to existing clients

Give me a number and a plan, not a lecture.
```

## Learning & skill development

### PW-11: What should I learn next?
```
I'm [N] months into my freelance practice. Current services: [LIST]. Current clients: [TYPE + COUNT].

My goals in 6 months:
[PASTE]

Recommend ONE skill or tool to invest 5 hours this week learning. Criteria:
- Directly improves an active client relationship OR opens a new service line
- Not shiny-object syndrome
- Trackable outcome (I can point to the work it unlocked)

Also: what am I currently trying to learn that I should stop learning because it's not moving the needle?
```

### PW-12: Feedback integration
```
A client just gave me this feedback:
[PASTE]

Reflect on it:
- What part is true (honest self-assessment)
- What part is their context (their preference vs. my skill gap)
- What's one concrete action I'll take
- What's one thing I should NOT change based on this (not everyone is the right client)

Be kind to me but honest. I need the real read.
```

## Systems & templates

### PW-13: Build a repeatable workflow for [TASK]
```
I keep doing [TASK] ad hoc each time. I want to systematize.

Typical steps (as I remember them):
[PASTE]

Turn into a reusable SOP:
- Trigger (when this starts)
- Step-by-step checklist (numbered, specific)
- Tools used per step
- Time estimate per step
- Common failure modes + how to avoid
- Quality checklist (what defines "done")

Format: copy-pasteable into Notion or a Google Doc.
```

### PW-14: Client onboarding SOP
```
Every new client I onboard, I do roughly the same things but in different orders. Let's systematize.

Create a client onboarding SOP covering:
- Day 0 (contract signed): welcome email + intake form
- Day 1-3: kickoff call + clarifications
- Day 4-7: first delivery or first check-in
- Day 14: mid-engagement check
- Day 30: retention / upsell touch

For each, include:
- What I do
- What I send (link to template)
- What I expect from them
- What I capture for my records

Format as a checklist I can duplicate per new client.
```

## Bookkeeping my own business

### PW-15: Monthly financial self-check
```
Here's my freelance income/expenses for [MONTH]:
- Income: [$X from N clients — list]
- Fixed expenses: [$ — software, insurance, etc.]
- Variable expenses: [$ — contractor hours, etc.]
- Time worked: [HOURS]

Analyze:
- Effective hourly rate for the month
- Profit margin
- Trend vs last month
- Anything unusual to flag
- One adjustment I should make for next month

Be honest if I worked too many hours for too little money — I need the real read.
```

---

## Self-management rules

- Your schedule is your product. Protect it.
- "Busy" is not an achievement. Output is.
- Review, don't ruminate. Post-mortems move you forward; replays trap you.
- Celebrate small wins. Your brain needs proof that the work is worth it.
